# EAِ خودکارِ قطب‌نما (متاتریدر ۵ → GitHub Gist → اپ)

معاملاتِ بسته‌شده‌ات را خودکار و **دقیق** (با استاپِ اولیه و ریسکِ واقعی) وارد ژورنال می‌کند.

**چرا این روش؟** رایگان است، **از ایران باز است** (برخلاف Google Apps Script که بلاک است)، و دادهٔ معاملاتت در یک **Gistِ خصوصی** می‌ماند. از همان توکنی که برای بک‌آپ داری استفاده می‌کند.

## زنجیره
```
متاتریدر (EA)  ──PATCH──▶  GitHub Gistِ خصوصی  ──GET──▶  اپ قطب‌نما
```

## گام‌ها (یک‌بار)

### ۱) توکنِ گیت‌هاب (اگر از قبل برای بک‌آپ نداری)
- GitHub → Settings → Developer settings → **Personal access tokens (classic)** → Generate.
- فقط scopeِ **`gist`** را تیک بزن. توکن را کپی کن.

### ۲) یک Gistِ خصوصی بساز
- برو به <https://gist.github.com> → یک فایل با نامِ `qotbnama-ea.json` و محتوای `[]` بساز → **Create secret gist**.
- از آدرسِ صفحه، **شناسهٔ Gist** را بردار (بخشِ بلندِ بعد از نام‌کاربری در URL). مثال: `gist.github.com/ali/`**`a1b2c3d4e5...`** → شناسه = `a1b2c3d4e5...`.

### ۳) EA را نصب کن
- فایلِ `Qotbnama.mq5` را در متاتریدر → پوشهٔ `MQL5/Experts/` بگذار (File → Open Data Folder).
- در MetaEditor کامپایل کن (F7).

### ۴) WebRequest را مجاز کن (مهم)
- متاتریدر → Tools → Options → **Expert Advisors** → تیکِ **Allow WebRequest for listed URL** → اضافه کن:
  ```
  https://api.github.com
  ```

### ۵) EA را روی یک چارت بگذار و ورودی‌ها را پر کن
- `InpToken` = توکنِ گیت‌هاب
- `InpGistId` = شناسهٔ Gist
- `InpFile` = `qotbnama-ea.json` (پیش‌فرض)
- `InpDays` = مثلاً ۹۰ (چند روزِ اخیر ارسال شود)
- تیکِ **Allow Algo Trading** روشن باشد.

### ۶) اپ را وصل کن
- در اپ → تبِ **معاملات** → پنلِ «وارد کردن خودکار» → بخشِ «اتصالِ خودکار» → این را بگذار:
  ```
  gist:شناسهٔ‌گیست/qotbnama-ea.json
  ```
  (مثال: `gist:a1b2c3d4e5.../qotbnama-ea.json`) — با همان توکنِ بک‌آپ خوانده می‌شود.
- «همگام‌سازی الان» را بزن. از این به بعد هر بار اپ باز شود، خودکار می‌آید.

## چه چیزی دقیق‌تر می‌شود؟
EA **استاپِ لحظهٔ ورود** و **ریسکِ واقعیِ دلاری** (فاصلهٔ استاپ × ارزشِ تیک × حجم) و **کمیسیون/سواپ** را می‌فرستد. پس R = سودِ خالص ÷ ریسکِ واقعی — دقیق، ترِیل‌پروف، و شاملِ کارمزد. (حالتِ گزارشِ HTML این‌ها را ندارد چون فقط «آخرین» استاپ را دارد.)

## عیب‌یابی
- **`WebRequest خطا: 4060`** یا `-1`: آدرسِ `https://api.github.com` را در گام ۴ اضافه نکرده‌ای.
- **HTTP 401**: توکن غلط است یا scopeِ `gist` ندارد.
- **HTTP 404**: شناسهٔ Gist اشتباه است.
- **در اپ چیزی نمی‌آید**: مطمئن شو توکنِ بک‌آپ در تنظیماتِ اپ ذخیره شده و همان حسابِ گیت‌هاب است.
- استاپِ اولیه فقط وقتی ثبت می‌شود که هنگامِ ورود، SL روی سفارش تنظیم شده باشد. اگر SL را بعد از ورود می‌گذاری، ریسکِ آن معامله دقیق نخواهد بود.

> نکته: `PATCH` در WebRequestِ نسخه‌های جدیدِ متاتریدر (build ۳۰۰۰+) پشتیبانی می‌شود. اگر بیلدت قدیمی است و کار نکرد، متاتریدر را به‌روز کن.

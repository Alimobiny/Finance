# قطب‌نما

اپلیکیشن شخصی مدیریت مالی، ژورنال معاملات و برنامهٔ روزانه — به‌صورت PWA نصب‌شونده روی گوشی و لپ‌تاپ، با همگام‌سازی داده از طریق Google Drive شخصی کاربر (بدون سرور یا دیتابیس اختصاصی).

جزئیات کامل معماری و نقشهٔ راه ساخت در فایل پلن پروژه ثبت شده است.

## توسعهٔ محلی

```bash
npm install
npm run dev
```

## بیلد نهایی

```bash
npm run build   # خروجی در dist/
npm run preview # پیش‌نمایش بیلد نهایی
```

## ساختار پروژه

```
src/
  app/        پوسته اصلی اپ، ناوبری، هدر
  components/ کامپوننت‌های عمومی و قابل‌استفادهٔ مجدد (ui/)
  features/   هر بخش اپ (dashboard, portfolio, trading, money, life, settings)
  store/      مدیریت وضعیت (Zustand)
  lib/        ابزارهای مشترک (فرمت عدد/تاریخ، همگام‌سازی Drive)
  types/      مدل داده مشترک کل اپ
```

## راه‌اندازی همگام‌سازی Google Drive

داده‌ها برای همگام‌شدن بین گوشی و لپ‌تاپ در یک فایل JSON داخل پوشهٔ خصوصی اپ در Google Drive شخصی‌ات ذخیره می‌شوند (نامرئی در فضای عادی Drive). تا این تنظیمات را انجام ندهی، اپ فقط به‌صورت محلی (localStorage) کار می‌کند — چیزی از دست نمی‌رود، فقط بین دستگاه‌ها سینک نمی‌شود.

این مراحل را فقط خودت (با اکانت گوگلت) می‌توانی انجام دهی:

1. به [Google Cloud Console](https://console.cloud.google.com/) برو و یک پروژهٔ جدید بساز (یا یکی موجود را انتخاب کن).
2. از منوی «APIs & Services» → «Library»، سرویس **Google Drive API** را پیدا و فعال کن.
3. به «APIs & Services» → «OAuth consent screen» برو:
   - نوع کاربر: **External**
   - وضعیت انتشار را روی **Testing** نگه دار (نیازی به بازبینی رسمی گوگل برای یک اپ شخصی نیست)
   - ایمیل گوگل خودت را به‌عنوان «Test user» اضافه کن
4. به «APIs & Services» → «Credentials» برو → «Create Credentials» → **OAuth client ID** → نوع **Web application**.
5. در بخش «Authorized JavaScript origins» این دو آدرس را (هرکدام در یک خط جدا) اضافه کن:
   - `http://localhost:5173` (برای توسعهٔ محلی)
   - آدرس نهایی GitHub Pages، مثلاً `https://USERNAME.github.io`
6. Client ID ساخته‌شده را کپی کن و در یک فایل به‌نام `.env.local` (که در گیت ذخیره نمی‌شود) این‌طور قرار بده:
   ```
   VITE_GOOGLE_CLIENT_ID=xxxxxxxx.apps.googleusercontent.com
   ```
7. برای نسخهٔ منتشرشده روی GitHub Pages، همین مقدار را به‌عنوان یک GitHub Actions secret/variable به نام `VITE_GOOGLE_CLIENT_ID` هم اضافه کن.

نکات مهم:
- چون اپ در حالت Testing می‌ماند، هر بار ورود یک صفحهٔ هشدار «Google این اپ را تأیید نکرده» با یک کلیک اضافه («ادامه») نشان می‌دهد — کاملاً عادی و صرفاً ظاهری است.
- ورود گوگل حدود ۱ ساعت اعتبار دارد و رفرش‌توکن ندارد (چون بک‌اندی وجود ندارد)؛ گاهی لازم است دوباره وارد شوی.
- در تنظیمات اپ، بخش «پشتیبان‌گیری دستی» همیشه در دسترس است — مستقل از Drive، برای دانلود/بازیابی دستیِ کل داده.

## قیمت لحظه‌ای دارایی‌ها (خودکار و رایگان)

قیمت‌های بازار ایران (دلار، تتر، سکه، طلای ۱۸) با یک GitHub Action به‌صورت دوره‌ای از `tgju.org` گرفته و در `public/prices.json` نوشته می‌شوند؛ اپ آن‌ها را same-origin می‌خواند (بدون مشکل CORS و بدون هیچ کلید در اپ). در صفحهٔ پرتفولیو دکمهٔ **«⟳ دریافت خودکار قیمت‌ها»** آخرین مقادیر را اعمال می‌کند.

- زمان‌بندی پیش‌فرض: هر ساعت (`.github/workflows/prices.yml`). برای نزدیک‌تر شدن به لحظه‌ای، cron را به `*/30 * * * *` تغییر بده.
- هیچ راه‌اندازی‌ای لازم نیست؛ خودکار کار می‌کند. اگر روزی خواستی منبع پایدارتری با کلید (مثل BrsApi) بگذاری، کافی است تابع `fromTgju` در `scripts/fetch-prices.mjs` را جایگزین کنی (کلید در Secrets می‌ماند، نه در اپ).
- اسکریپت مقاوم است: اگر منبع خطا داد، آخرین مقدار معتبر حفظ می‌شود و هرگز با صفر بازنویسی نمی‌شود.

## خلاصهٔ اخبار کانال تلگرام (خودکار و رایگان)

کارت «خلاصهٔ اخبار هفته» در داشبورد، ۱۰ خبر مهمِ یک هفتهٔ اخیرِ یک **کانال عمومی تلگرام** را نشان می‌دهد. یک GitHub Action (`.github/workflows/news.yml`) پیام‌ها را از `t.me/s/<channel>` می‌خواند، با **GitHub Models** (رایگان، داخل خودِ Action و بدون کلید خارجی) خلاصه می‌کند و در `public/news.json` می‌نویسد.

راه‌اندازی (فقط یک مرحله):

1. به ریپو → **Settings → Secrets and variables → Actions → Variables** برو و یک متغیر بساز:
   - نام: `TELEGRAM_CHANNEL`
   - مقدار: یوزرنیم کانال عمومی (مثلاً `tgju` — بدون `@` و بدون `https://t.me/`).
2. همین بس است. برای اجرای فوری (بدون منتظر ماندن تا فردا صبح)، از تب **Actions** ورک‌فلوی «Update news» را با **Run workflow** دستی اجرا کن.

نکات:
- کانال باید **عمومی** باشد (کانال خصوصی صفحهٔ `t.me/s` ندارد).
- زمان‌بندی پیش‌فرض روزی یک‌بار (~۸ صبح تهران) است؛ در `news.yml` قابل تغییر.
- اگر خلاصه‌سازی به هر دلیل شکست بخورد، همان آخرین پیام‌ها به‌صورت خام نمایش داده می‌شوند تا کارت خالی نماند.

## وضعیت ساخت

این پروژه در چند مرحله (Milestone) ساخته می‌شود؛ داشبورد، پرتفولیو، معاملات (ژورنال/چک‌لیست/ماشین‌حساب)، مدیریت مالی، برنامهٔ روزانه، همگام‌سازی Google Drive و تنظیمات آماده‌اند. مرحلهٔ بعدی: راه‌اندازی گیت‌هاب/دیپلوی خودکار و پرداخت نهایی (تست آفلاین، آیکون‌ها، واکنش‌گرایی موبایل).
